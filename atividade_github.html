GIT HUB

GitHub é uma plataforma de hospedagem de código-fonte e arquivos com controle de versão usando o Git.
Ele permite que programadores, utilitários ou qualquer usuário cadastrado na plataforma contribuam em projetos privados e/ou Open Source de qualquer lugar do mundo.



Branch

Uma branch é uma linha de desenvolvimentoo que permite que você trabalhe em diferentes versões de um projeto simultaneamente. Quando você cria uma branch, você está essencialmente fazendo uma cópia do código em um determinado ponto no tempo. Isso ppermite que você faça alterações, adicione novos recursos ou corrija bugs sem afetar a versão principal do projeto (geralmente chamada de "main" ou "master").

Como funciona o Branch:

Quando você cria um branch, você pode fazer alterações e commits de forma isolada. Uma vez que as alterações estejam prontas e testadas, você pode mesclar (ou "merge") esse branch de volta ao branch principal, integrando suas modificações.
Isso ajuda a manter o código organizado e facilita a colaboração entre diferentes desenvolvedores, pois cada um pode trabalhar em suas próprias tarefas sem interferir no trabalho dos outros.



Para o que é utilizado?

O branch no Git serve para criar uma linha de desenvolvimento separada dentro de um repositório. Isso permite que você trabalhe em novas funcionalidades, correções de bugs ou experimentos sem afetar a versão principal do código, geralmente chamada de "main" ou "master".
Com os branches, você pode desenvolver e testar suas alterações de forma isolada e, quando estiver satisfeito, pode mesclar (merge) essas alterações de volta ao branch principal. Isso facilita a colaboração entre diferentes desenvolvedores e ajuda a manter o código organizado e estável.

Como criar um branch

1. Abra o terminal:



Navegue até o diretório do seu projeto.

2. Verifique o branch atual:


Use o comando git branch para ver qual branch você está.


3. Crie um novo branch:


Execute o comando mais um git branch nome-do-branch para criar um novo branch.


4. Mude para o novo branch:


Use git checkout nome-do-branch para mudar para o branch que você acabou de criar. Você também pode combinar esses dois passos usando o git checkout -b nome-do-branch.


Como gerenciar os branches?

1. Listar branches:


Use git branch para ver todos os branches disponíveis.

2. Mudar entre branches:


Use git checkout nome-do-branch para mudar para um branch existente.

3. Mesclar branches:


Quando você terminar de trabalhar em um branch e quiser integrar as mudanças ao branch principal (geralmente chamado de main ou master), primeiro mude para o branch principal com git checkout main e depois use git merge nome-do-branch.

4. Excluir um branch:


Se você não precisar mais de um branch, pode excluí-lo com git branch -d nome-do-branch
